Operating Systems Terminology

TERM

DEFINITION
operating system The software that controls the overall operation of the computer including file management, the user interface, and the program execution environment.
job queue Mass storage wait for jobs in batch proccessing systems.
FIFO First-in, First out. The order of a queue.
interactive processing Remote terminals that allowed a program being executed to carry on a dialogue with the user.
time-sharing A feature that allowed new operating systems to provide service to multiple users at the same time.
multiprogramming A technique for implementing time-sharing (see above) where time is divided into intervals and then the execution of each job is restricted to only one interval at a time.
application software Consists of the programs for performing tasks particular to the machine's utilization.
system software Performs tasks that are common to computer systems in general.
utility software A system software category that consists of software units collectively
graphical user interface (GUI) A user interface in which objects can be manipulated and are represented as icons.
shell An older user interface that communicated with users through textual messages using a keyboard and monitor screen.
kernal The internal part of an operating system
device driver A component of the kernal which are the software units that communicate with the controllers (or at times, directly with the peripheral devices) to carry out operations on the peripheral devices attached to the machine.
file manager Coordinates the use of the machine's mass storage facilities.
memory manager Coordinates the machine's use of main memory.
virtual memory Space created by the memory manager to allow the computer to fuction as if it had additional memory.
boot strapping A procedure that transfers the operating system from mass storage (where it is permanently stored) into main memory.
ROM Read-only Memory is special nonvolatile memory that can be read but not altered
deadlock A resource allocation problem in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
spooling A technique of holding data for output at a later but more convenient time.
super user (or administrator) A highly privileged account/access to the operating system
multi-factor authentication Several methods of security authorization meant to provide users more protection
